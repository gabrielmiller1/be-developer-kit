Problemas comuns em pacotes Adobe

Veja abaixo explicações e soluções para os principais problemas detectados em pacotes Adobe.

Índice:
  • Filtro muito genérico
  • Path proibido no filtro
  • Filtro fora do escopo do projeto
  • Filtro duplicado
  • Propriedade obrigatória ausente
  • Versão inválida
  • Configuração OSGi encontrada
  • Propriedade perigosa (admin)
  • Template fora do projeto
  • Design path fora do projeto
  • Mixin não permitido
  • Nome da query GraphQL inválido
  • Query GraphQL com wildcard
  • Query GraphQL referencia outro projeto
  • Múltiplas queries no arquivo
  • Arquivo indesejado
  • Estrutura obrigatória ausente
  • Arquivo não encontrado
  • Arquivo muito grande
  • Nome do pacote inválido
  • Muitos arquivos no pacote
  • Filter.xml ausente
  • Properties.xml ausente
  • Conteúdo JCR ausente
  • Configurações OSGi encontradas
  • Imagem sem atributo alt
  • Link sem descrição
  • Contraste insuficiente

Filtro duplicado

O mesmo path está sendo filtrado múltiplas vezes no filter.xml.

Como corrigir:
  • Remova filtros duplicados, mantendo apenas uma entrada por path.

Propriedade obrigatória ausente

Propriedades essenciais estão faltando no arquivo properties.xml.

Como corrigir:
  • Adicione as propriedades obrigatórias (name, group, version) no properties.xml.
<properties>
  <entry key="name">meu-pacote</entry>
  <entry key="group">meu.grupo</entry>
  <entry key="version">1.0.0</entry>
</properties>

Versão inválida

A versão do pacote não segue o padrão Semantic Versioning (semver).

Como corrigir:
  • Use o formato MAJOR.MINOR.PATCH (ex: 1.0.0) para a versão.

Configuração OSGi encontrada

Configurações OSGi não devem ser incluídas em pacotes de conteúdo.

Como corrigir:
  • Remova arquivos de configuração OSGi dos pacotes de conteúdo.

Propriedade perigosa (admin)

O conteúdo foi modificado pelo usuário admin, o que pode causar problemas de segurança.

Como corrigir:
  • Remova ou altere a propriedade cq:lastModifiedBy="admin".

Template fora do projeto

O template referenciado não pertence ao projeto atual.

Como corrigir:
  • Use apenas templates do projeto atual ou garanta que o template referenciado existe.

Design path fora do projeto

O caminho de design não pertence ao projeto atual.

Como corrigir:
  • Use apenas caminhos de design do projeto atual.

Mixin não permitido

Mixins JCR além de mix:versionable não são permitidos.

Como corrigir:
  • Remova mixins não permitidos, mantendo apenas mix:versionable se necessário.
<jcr:mixinTypes>
  <jcr:mixinType>mix:versionable</jcr:mixinType>
</jcr:mixinTypes>

Nome da query GraphQL inválido

O nome da query GraphQL deve seguir o padrão do projeto.

Como corrigir:
  • Renomeie o arquivo para começar com o nome do projeto seguido de underscore.
  • Exemplo: meu-projeto_minha-query.graphql

Query GraphQL com wildcard

Queries GraphQL não podem conter wildcards (*).

Como corrigir:
  • Remova wildcards e seja específico nos campos da query.

Query GraphQL referencia outro projeto

Queries GraphQL devem referenciar apenas conteúdo do projeto atual.

Como corrigir:
  • Ajuste os paths para referenciar apenas conteúdo do projeto atual.

Múltiplas queries no arquivo

Arquivos GraphQL devem conter apenas uma query.

Como corrigir:
  • Separe cada query em um arquivo diferente.

Arquivo indesejado

Arquivos temporários ou de sistema foram incluídos no pacote.

Como corrigir:
  • Remova arquivos como .DS_Store, Thumbs.db, .gitkeep, etc.

Estrutura obrigatória ausente

Estruturas essenciais do pacote Adobe estão faltando.

Como corrigir:
  • Garanta que META-INF/vault/ e jcr_root/ estejam presentes.

Arquivo não encontrado

O arquivo especificado não foi encontrado no sistema.

Como corrigir:
  • Verifique se o caminho do arquivo está correto.

Arquivo muito grande

O tamanho do pacote excede o limite permitido.

Como corrigir:
  • Reduza o tamanho do pacote para no máximo 200MB.
  • Divida em pacotes menores se necessário.

Nome do pacote inválido

O nome do pacote não segue o padrão Adobe AEM.

Como corrigir:
  • Use o formato: [a-z0-9-]+(content|apps|conf)-YYYYMMDD-*.zip
  • Exemplo: meu-projeto-content-20240101-v1.0.0.zip

Muitos arquivos no pacote

O pacote contém um número excessivo de arquivos.

Como corrigir:
  • Reduza o número de arquivos ou divida em pacotes menores.

Filter.xml ausente

O arquivo filter.xml é obrigatório em pacotes Adobe.

Como corrigir:
  • Crie o arquivo META-INF/vault/filter.xml com os filtros apropriados.

Properties.xml ausente

O arquivo properties.xml é obrigatório em pacotes Adobe.

Como corrigir:
  • Crie o arquivo META-INF/vault/properties.xml com as propriedades do pacote.

Conteúdo JCR ausente

Não foi encontrado conteúdo na pasta jcr_root.

Como corrigir:
  • Adicione conteúdo válido na pasta jcr_root do pacote.

Configurações OSGi encontradas

Pacotes de conteúdo não devem incluir configurações OSGi.

Como corrigir:
  • Remova todas as configurações OSGi do pacote.

Filtro muito genérico

Filtros muito genéricos no filter.xml podem incluir conteúdo indesejado ou fora do escopo do projeto.

Como corrigir:
  • Use paths específicos como /content/nome-do-projeto, /apps/nome-do-projeto ou /conf/nome-do-projeto.
  • Evite filtros genéricos como /content, /apps ou /conf.
<filter root="/content/meu-projeto"/>

Path proibido no filtro

Paths como /libs e /etc não devem ser incluídos em pacotes de conteúdo, pois são áreas do sistema.

Como corrigir:
  • Remova filtros que apontem para /libs ou /etc.
  • Use apenas paths específicos do seu projeto.

Filtro fora do escopo do projeto

O filtro aponta para um path que não pertence ao projeto atual.

Como corrigir:
  • Verifique se o path do filtro corresponde ao nome do projeto extraído do nome do pacote.
  • Exemplo: se o pacote é "meu-projeto-content-20240101-v1.0.0.zip", use /content/meu-projeto.

Imagem sem atributo alt

O atributo alt em imagens é essencial para acessibilidade, pois descreve o conteúdo da imagem para pessoas que utilizam leitores de tela.

Como corrigir:
  • Sempre adicione o atributo alt em todas as imagens.
  • Descreva brevemente o conteúdo ou função da imagem.
  • Se a imagem for decorativa, use alt="".
<img src="logo.png" alt="Logo da empresa" />

Link sem descrição

Links devem ter descrições claras para que todos os usuários entendam seu propósito, especialmente quem utiliza leitores de tela.

Como corrigir:
  • Evite textos genéricos como "clique aqui".
  • Descreva o destino ou ação do link.
<a href="/contato">Fale conosco</a>

Contraste insuficiente

O contraste entre texto e fundo deve ser suficiente para garantir a leitura por todos os usuários, inclusive pessoas com baixa visão.

Como corrigir:
  • Utilize ferramentas para checar o contraste.
  • Garanta contraste mínimo de 4.5:1 para textos normais.
// Exemplo de bom contraste
Cor de fundo: #ffffff
Cor do texto: #222222
